Overview
Style is a design system that defines the interaction standards, design patterns, and guidelines for all ZALORA products. It was my job to provide aid from the development perspective and creatively direct the creation and documentation of the design system. I worked with Pramiti (UX Manager, Interaction), Eugene (UX Manager, Research), Sheila (UI Designer) and, Doro (Product Manager) on the project.
Deliverables
  • ZALORA/Style (CSS component library)
  • Style-react (React component library)
  • Documentation Portal

My Role

As a frontend developer on the team, I bring a frontend focus with HTML & CSS knowledge, seasoned skills to establish conventions and tool building. My responsibilities range from reviewing and auditing the as-is design and implementation, providing aid with frontend focus to ensure the design is scalable and reusable, contributing to the design system documentation.

The Process

Strategy

System review

We started the project by assessing as-is design assets, code libraries, documentation.

Mission & Vision

The design system is not only aimed to provide a joyful, enriching, unified experience to the customers but also ensures a low cost of changes and a balance between creativity and efficiency.

Design, Code & Doc

We delivered the components/elements progressively over sprints. We started by design and building the visual style (e.g., color, typography, and space), which was followed by UI components (e.g., Button, Tabs, Labels).

Data-driven Design

I looked for tracking data in Google Analytics that can be used to inform the design decisions. Based on the traffic from different platforms, we decided to adopt a mobile-first approach. We defined the breakpoints for the grid system by referencing the distribution of the screen sizes.

Implementation

Due to the current implementation of the mobile website and the desktop website, we implemented two libraries: ZALORA/Style and ZALORA/Styleguide.

ZALORA/Style

This library was built by CSS and HTML. It defined the appearance of components and the animations.

ZALORA/Style-react

This library defined the look of the components by referencing ZALORA/Style. It also defined the behaviors assisted by ReactJs. It mainly served for the mobile platform.

Documentation

There were two kinds of documentation being provided: the design guidelines, and the distribution guides.

Design Guidelines

The design guidelines include the general guides (e.g., principles of ZALORA product design, ZALORA's tone of voice that governed the whole design system), as well as the docs for individual components (e.g., the spec, usage in code and design, and suggested/anti patterns).

Distribution Guides

The distribution guides is for developers who wants to distribute to the library. It includes the library structure, the conventions, tooling, etc.

Workshops & Talks

In order to get more people to use our new design system, we held workshops and talks to market our design system. Our participants were not limited to developers, PMs, and designers. We also invited the QAs and content team marketing department.


Apart from holding workshops and talks to introduce the design system, as a developer, I also implemented a playground as part of the documentation portal to encourage people to try the new design system without bothering setting up the development environment.
